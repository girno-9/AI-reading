# AI 读书

AI 读书是一个本地优先的读书配音项目。当前版本已开发完成第 1-2 阶段，并额外支持 EPUB 导入和章节插画预览：导入书稿、自动切分章节和段落、基础识别对白角色，并在网页中手动编辑段落和角色。

后续会拆成两个相互配合的方向：独立的“声音工坊”负责把角色音频素材整理成可复用的角色声音文件；AI 读书主流程负责把书稿整理成朗读工程，在第 3-4 阶段导入角色声音文件、标记情绪、逐段试听、批量生成和导出音频。

## 声音工坊：独立项目定位

声音工坊不直接负责某一本书的分章、分段或导出音频，它是一个独立的声音资产生产工具。它的输入是某个角色或某类声音的音频素材库，输出是可以被 AI 读书第 3-4 阶段使用的“角色声音包”。

声音工坊需要完成的事情：

- 导入角色音频素材：支持批量导入 `.wav`、`.mp3`、`.m4a`、`.flac`、`.ogg` 等常见音频。
- 建立角色声音档案：记录角色名、声音名、适用范围、音色描述、默认语速/情绪等信息。
- 管理训练样本：每条样本需要保存音频、样本文本/转写、情绪标签、备注、是否可用于训练。
- 标注情绪和风格：例如平静、开心、愤怒、悲伤、紧张、旁白、低声、急促等；具体标签后续可以按模型需要调整。
- 筛选训练素材：区分“可训练”“仅参考”“弃用”，避免噪声、错字、多人声、背景音乐等素材污染训练。
- 生成训练清单：导出模型训练需要的 metadata，例如音频文件、对应文本、情绪标签、角色标识。
- 对接本地声音训练/克隆工具：后续可以接 GPT-SoVITS 或其他本地方案，把素材训练成角色声音。
- 导出角色声音包：包含角色声音元数据、训练结果路径或模型标识、可用情绪、示例音频、必要配置。

建议的角色声音包 JSON 结构方向：

```json
{
  "phase": "voice_workshop_character_package",
  "export_version": 1,
  "character_name": "张三",
  "voice_name": "张三-青年男声",
  "description": "清亮、语速中等，适合主角对白",
  "supported_emotions": ["平静", "开心", "紧张"],
  "training_samples": [
    {
      "audio_data_url": "data:audio/wav;base64,...",
      "transcript": "你好，我叫张三。",
      "emotion": "平静",
      "note": "干净单人声",
      "usable_for_training": true
    }
  ],
  "voice_profile": {
    "engine": "gpt-sovits",
    "model_path": "",
    "config_path": "",
    "reference_audio_data_url": ""
  }
}
```

AI 读书主流程与声音工坊的关系：

```text
声音工坊：音频素材 -> 角色声音包
第 1-2 阶段：书稿 -> 分章分段角色 JSON
第 3 阶段：书稿 JSON + 角色声音包 -> 情绪标注和逐段试听 -> 朗读工程 JSON
第 4 阶段：朗读工程 JSON -> 批量生成音频 -> 章节/整书音频导出
```

## How to use

### 最简单的一键启动

1. 确认电脑已经安装 Python 3.12 或更新版本、Node.js 20 或更新版本。
2. 双击运行：

```text
scripts\start_phase_1_2.bat
```

3. 脚本会自动检查并安装依赖，然后分别打开后端和前端窗口。
4. 浏览器打开：

```text
http://127.0.0.1:5173
```

5. 在左侧输入书名，把 `.txt` 或 `.md` 内容粘贴到“正文”里，然后点击“导入文本”。
6. 如果要导入 EPUB，在同一个导入区选择 `.epub` 文件，然后点击“导入 EPUB”。
7. EPUB 导入时页面会显示“正在解析 EPUB...”，请等待解析完成，不要重复点击。
8. 如果 EPUB 导入失败，先查看“AI Reading Backend”这个后端 CMD 窗口，里面会显示具体错误和 traceback。
9. 导入后可以在页面里查看章节、章节插画、编辑段落、切换角色、新增角色、重命名角色。
10. 在右侧“角色管理”里，可以用箭头调整角色顺序，用置顶按钮把常用角色放到第一位；删除角色前会二次确认，正文段落里原本使用该角色的内容会自动改为当前置顶角色。
11. 如果章节切分不理想，点击章节标签下方的“全文分章节”，在弹窗里用 `@章节名: 名称` 手动标记每个章节；如果章节里有插画，会出现 `@插画: img_001` 这样的短编号，移动整行即可调整插画所在章节；确认后会替换整本书所有章节和段落。
12. 编辑完成后，在右侧“导出”区域下载整本书或当前章节，支持 JSON 和 TXT 两种格式。
13. 如果之后要继续修改已导出的结果，在右侧“导出”区域选择之前导出的整本 JSON，然后点击“导入为新书”；系统会新建一本副本，不覆盖原书。
14. 如果只想单独启动后端并查看错误日志，可以双击运行 `scripts\start_backend_debug.bat`。
15. 如果要清空网页里的已导入书籍，点击“书籍”标题旁边的删除按钮；系统会二次确认，并删除本地数据库和插画缓存。

### 使用示例文本

1. 打开 `examples\sample_book.txt`。
2. 复制里面的全部文字。
3. 粘贴到网页左侧“正文”输入框。
4. 书名可以填写“示例书稿”。
5. 点击“导入书稿”，查看章节切分和角色识别结果。

### 每个阶段的一键文件

| 阶段 | 一键文件 | 当前作用 |
| --- | --- | --- |
| 第 1-2 阶段 | `scripts\start_phase_1_2.bat` | 启动当前已开发的文本导入、分段、角色识别和手动编辑 Web 应用 |
| 后端调试 | `scripts\start_backend_debug.bat` | 单独启动后端，并在 CMD 窗口显示详细错误日志 |

| 第 4 阶段 | `scripts\start_phase_4_tts.bat` | 当前为占位提示，后续用于启动音频生产台：批量生成段落音频、拼接章节音频并导出 |
| 第 5 阶段 | `scripts\start_phase_5_github.bat` | 当前为占位提示，等发布流程确认后用于辅助 GitHub 发布 |

## 当前功能

- 导入 `.txt` / `.md` 文本内容。
- 导入 `.epub` 文件，并提取 EPUB 内的章节正文和插图。
- 按中文章节标题和空行切分章节、段落。
- 自动识别常见说话人格式，例如 `张三说：“你好。”`、`李四问道：“你是谁？”`。
- 未识别角色的段落默认归为“旁白”；只有对白但没有说话人时归为“未知角色”。
- 在网页中编辑段落文本、切换段落角色、新建角色、重命名角色、删除角色、调整角色上下顺序，并支持一键置顶角色。
- 删除角色时会二次确认；如果正文段落正在使用被删除的角色，这些段落会自动改为当前置顶角色。至少会保留一个角色，不能删空。
- 当前章节支持“一键保存本章节”，只提交有改动的段落，并在状态栏提示成功/失败数量。
- 当前章节支持“重新分段”：
  - 点击章节标签下面的“重新分段”按钮。
  - 在弹窗里用 `@段落名: 名称` 标注段落名。
  - 用 `@正文开始` / `@正文结束` 包住会显示且会朗读的正文。
  - 用 `@显示开始` / `@显示结束` 包住只显示、不朗读的内容。
  - 确认后会替换本章节所有段落；文本完全相同的旧段落会尽量保留原来的手动角色。
- 整本书支持“全文分章节”：
  - 点击“重新分段”旁边的“全文分章节”按钮。
  - 在弹窗里用 `@章节名: 名称` 标记每个章节。
  - 插画会显示为 `@插画: img_001` 这样的短编号，可以把这一整行移动到任意章节。
  - 弹窗右侧会显示插画编号库，可以一边看缩略图，一边确认 `img_001` 应该放在哪个章节。
  - 删除正文里的 `@插画: img_001` 不会删除原图；插画仍保存在本书的插画编号库里，之后重新输入这个编号可以恢复。
  - 如果输入不存在的编号，例如 `@插画: img_999`，系统会提示找不到该插画编号，避免静默丢图。
  - 一章里可以放多行 `@插画:`，系统会按出现顺序保存为该章节的多张插画。
  - `@插画:` 行只用于恢复插画位置，不参与正文分段、不朗读，也不会出现在 TXT 导出正文里。
  - 每个章节名下面直接写该章全文，下一次 `@章节名` 会开始新章节。
  - 该功能只负责重新划分章节；章节内部会自动分段，段落细节可以再用当前章节的“重新分段”修改。
  - 确认后会替换整本书所有章节和段落，不能恢复；文本完全相同的旧段落会尽量保留原来的手动角色。
- 在章节顶部显示插画区域；当前章节有插图时显示插图，没有插图时显示空状态。
- 导出整本书或当前章节：
  - JSON：包含章节、段落、角色、插画 URL 和插画 `data_url`，给下一阶段声音分配/TTS 使用。
  - TXT：包含章节标题和 `[角色] 文本`，方便人工校对。
- 回导已导出的整本 JSON：
  - 只支持 JSON，不支持 TXT 回导。
  - 回导后会新建一本书，标题后追加“（回导）”。
- 清空缓存：
  - 会删除所有已导入书籍、章节、段落、角色和本地插画缓存。
  - 不会影响你已经下载到电脑上的导出 JSON。
  - 因为整本 JSON 导出会内嵌插画 `data_url`，所以清空后仍可用该 JSON 回导并恢复插画。
- 使用 SQLite 保存数据，刷新页面后内容仍然存在。

## 项目结构

```text
backend/    FastAPI 后端、SQLite 数据库、文本解析和角色识别
frontend/   React + Vite 前端
examples/   示例书稿
scripts/    每个阶段的一键启动或提示脚本
```

## 手动启动后端

```powershell
cd backend
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
uvicorn app.main:app --reload --host 127.0.0.1 --port 8000
```

后端接口地址：

```text
http://127.0.0.1:8000
```

## 手动启动前端

另开一个终端：

```powershell
cd frontend
npm.cmd install
npm.cmd run dev
```

如果你的 PowerShell 允许执行 npm 脚本，也可以使用 `npm install` 和 `npm run dev`。

前端默认地址：

```text
http://127.0.0.1:5173
```

如需修改后端地址，可以设置：

```powershell
$env:VITE_API_BASE="http://127.0.0.1:8000"
npm.cmd run dev
```

cd backend
pytest
